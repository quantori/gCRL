Partial-MCC Permutation Experiments — Detailed Explanation
===============================================================

Function
--------
`run_partial_mcc_perm_experiments(adata, embeddings, ...)`

Purpose
-------
Quantify how strongly a set of community eigengenes (A) aligns with a provided cell-level
embedding matrix (B) using **partial-MCC optimization**, and assess significance by
constructing a **permutation-based null** where TF-to-community assignments are shuffled.

High-Level Steps
----------------
1) **Compute real eigengenes A (unpermuted):**
   - Calls `compute_eigengenes(...)` on the *current* `adata`.
   - Stores the result in `adata.obsm["X_comm_eig"]` (overwrites if present).
   - Extracts `A_real` and checks that it is row-aligned with `embeddings` B.

2) **Optimize partial-MCC on the real A (multiple seeds):**
   - For `n_real_seeds` different random seeds, calls `optimize_partial_mcc(A_real, B, ...)`.
   - Collects a vector of scores: `scores_real` with shape `(n_real_seeds,)`.

3) **Build the null via TF-community permutations:**
   - Saves original `adata.var[community_col]` to restore later.
   - Repeats for `n_permutations`:
     a) Randomly permute **only TF genes'** community labels in-place
        (`_permute_tf_communities_inplace`). The set of TF genes and the multiset of labels are preserved;
        labels are reshuffled among TFs.
     b) Recompute eigengenes under the permuted labels, getting `A_perm`.
     c) Run `optimize_partial_mcc(A_perm, B, ...)` for `n_perm_seeds` seeds and store the scores.
   - This yields a matrix `scores_perm` of shape `(n_permutations, n_perm_seeds)`.

4) **Restore original communities:**
   - Returns `adata.var[community_col]` to its original state.

5) **Optional visualization:**
   - If `save_density_path` is provided, saves a PNG showing kernel-density histograms for
     `scores_real` and all permuted scores (flattened).

Inputs and Assumptions
----------------------
- **AnnData (`adata`)**
  * `adata.X`: normalized/log1p expression.
  * `adata.var["kind"]`: must mark genes as "TF" or "TG" (case-insensitive check upstream).
  * `adata.var[community_col]`: categorical or label-like community assignments.
  * Note: `compute_eigengenes` **mutates** `adata` by writing `A` to `obsm["X_comm_eig"]`.

- **Embeddings (`embeddings`)**
  * A 2D NumPy array `B` of shape `(n_cells, pB)`, typically the latent space of an autoencoder.
  * Must be **row-aligned** with `adata.obs_names`. The function enforces `A.shape[0] == B.shape[0]`.

- **Eigengene configuration (forwarded to `compute_eigengenes`)**
  * `community_col`: column in `adata.var` defining communities.
  * `reference_query`: used within eigengene computation (e.g., define unperturbed reference).
  * `mode`: passed through (e.g., "by_cell_type" vs "global"); controls how eigengenes are aggregated.
  * `cell_type_col`: used when `mode` requires cell-type stratification.

- **Experiment sizes**
  * `n_real_seeds`: number of MCC optimizations on the real A.
  * `n_permutations`: number of TF-label shuffles for the null.
  * `n_perm_seeds`: number of MCC runs per permutation.

- **Optimizer parameters**
  * `lr`, `steps`, `device` are forwarded to `optimize_partial_mcc`.
  * `device` can be "cpu", "cuda", or None to auto-select.

- **Reproducibility**
  * `master_seed` seeds an RNG used to draw all per-run seeds.
  * Seeds are sampled in `[0, 2**31 - 1)`. Both real and permuted runs use independent seeds derived from this master stream.

Key Internal Functions
----------------------
- `_assert_row_alignment(A, B)`: raises an error if A and B differ in number of rows.
- `_permute_tf_communities_inplace(adata, community_col, seed)`:
  * Identifies TF rows as `adata.var["kind"].str.upper() == "TF"`.
  * Shuffles the existing community labels **among TFs only** using the given seed.
  * Leaves TG labels unchanged and preserves label frequencies.

Returned Values
---------------
A dictionary with two arrays:
- `scores_real`: shape `(n_real_seeds,)`. Partial-MCC values from the real (unpermuted) eigengenes.
- `scores_perm`: shape `(n_permutations, n_perm_seeds)`. Partial-MCC values under the null created by TF-label shuffles.

Interpretation
--------------
- **Higher partial-MCC** implies a stronger alignment between eigengenes `A` and embeddings `B` under the optimizer’s objective.
- Significance can be assessed by comparing the distribution of `scores_real` to the **null** distribution
  (flattened `scores_perm`) via empirical p-values, e.g.:
    p̂ = (1 + count(flat_perm >= median(scores_real))) / (1 + len(flat_perm))
  or by comparing means/quantiles depending on design.
- The permutation scheme specifically tests whether **TF-to-community structure** explains the alignment;
  TG labels remain as-is, so the null targets the TF structure of communities.

Data Mutability and Restoration
-------------------------------
- The function mutates:
  * `adata.obsm["X_comm_eig"]` (each time eigengenes are recomputed).
  * `adata.var[community_col]` during permutations.
- Original community labels are **fully restored** at the end.

Device and Performance Notes
----------------------------
- `optimize_partial_mcc` may use GPU if `device="cuda"`.
- Eigengene computation is typically CPU-bound; overall runtime scales with:
  * `n_real_seeds`
  * `n_permutations × n_perm_seeds`
  * cost of `compute_eigengenes` and the optimizer `steps`

Common Pitfalls and Checks
--------------------------
- **Row alignment:** If `A.shape[0] != B.shape[0]`, an error is raised.
- **Missing annotations:** `adata.var` must contain `kind` and `community_col`.
- **Re-entrancy:** Because `adata` is mutated, avoid concurrent calls that share the same `adata` instance
  without proper synchronization. This function restores `community_col` but not previous `obsm["X_comm_eig"]` contents beyond its final state.

Optional Plot
-------------
If `save_density_path` is a valid file path (e.g., "mcc_density.png"):
- Saves a 7×5 inch PNG with overlaid histograms (density-normalized) of `scores_real` and all
  null scores. This is a quick diagnostic; for publication-quality figures, consider KDE overlays,
  transparent fills, and reporting exact summary statistics.

Typical Usage
-------------
```
res = run_partial_mcc_perm_experiments(
    adata,
    embeddings=Z,                     # shape (n_cells, pB), row-aligned to adata.obs
    community_col="community",
    reference_query='intervention == "unperturbed"',
    mode="by_cell_type",              # or "global" depending on your eigengene routine
    cell_type_col="cell_type",
    n_real_seeds=100,
    n_permutations=100,
    n_perm_seeds=10,
    lr=1e-2,
    steps=500,
    device=None,                      # auto-select
    master_seed=42,
    save_density_path=None
)
scores_real = res["scores_real"]
scores_perm = res["scores_perm"]
```

Extension Ideas
---------------
- **Parallelization:** run permutations in parallel (carefully managing RNG streams).

Link to Upstream Functions
--------------------------
- `compute_eigengenes`: computes `A` and stores it in `adata.obsm["X_comm_eig"]`. Parameters like
  `reference_query`, `mode` (“global” vs “by_cell_type”), and `cell_type_col` are **forwarded** as-is.
- `optimize_partial_mcc`: given `(A, B)`, optimizes a partial MCC alignment objective. The function returns
  `(score, params, history)`; in this experiment runner, only `score` is recorded.
