# Plan: Adding Cell-Type Conditioning to gCRL-VAE

Written 2026-04-30. Describes the changes needed to implement conditional VAE behaviour
(cell-type one-hot concatenation in encoder and decoder) starting from the current code.

Cell-type conditioning is currently a no-op: set_ct_index() exists for API compatibility
but ct_dim is always 0 and the encoder/decoder never see any cell-type information.
The plan below adds it as a first-class feature while keeping all existing signatures.

---

## 1. GCRLVAEConfig  (src/gcrl/models/gcrl_vae.py)

Add one field:

    ct_dim: int = 0    # 0 = no conditioning; >0 = number of cell types

No other config changes needed.

---

## 2. GCRLVAE.__init__  (src/gcrl/models/gcrl_vae.py)

The encoder input and decoder input must widen when ct_dim > 0.

Current:
    self.fc1 = nn.Linear(cfg.input_dim, cfg.hidden_enc)
    self.d1  = nn.Linear(cfg.z_dim, 128)

Change to:
    enc_in = cfg.input_dim + (cfg.ct_dim if cfg.ct_dim > 0 else 0)
    self.fc1 = nn.Linear(enc_in, cfg.hidden_enc)

    dec_in = cfg.z_dim + (cfg.ct_dim if cfg.ct_dim > 0 else 0)
    self.d1  = nn.Linear(dec_in, 128)

Add storage for the cell-type index (already started as a no-op):
    self.ct_index: Optional[Dict[str, int]] = None

---

## 3. GCRLVAE.set_ct_index  (src/gcrl/models/gcrl_vae.py)

Replace the current no-op with:

    def set_ct_index(self, ct_names):
        if isinstance(ct_names, dict):
            self.ct_index = dict(ct_names)
        else:
            assert len(ct_names) == self.cfg.ct_dim
            self.ct_index = {n: i for i, n in enumerate(ct_names)}

Add a helper to build a one-hot vector from a cell-type name:

    def _ct_one_hot(self, cell_type: str, B: int) -> Optional[torch.Tensor]:
        if self.cfg.ct_dim <= 0 or self.ct_index is None:
            return None
        idx = self.ct_index.get(str(cell_type))
        if idx is None:
            raise KeyError(f"cell_type '{cell_type}' not in ct_index")
        ct = torch.zeros(B, self.cfg.ct_dim, device=self._device())
        ct[:, idx] = 1.0
        return ct

---

## 4. GCRLVAE.encode  (src/gcrl/models/gcrl_vae.py)

Add ct parameter:

    def encode(self, x: torch.Tensor,
               ct: Optional[torch.Tensor] = None):
        if self.cfg.ct_dim > 0 and ct is not None:
            x = torch.cat([x, ct], dim=-1)
        h = F.leaky_relu(self.fc1(x), 0.2)
        return self.fc_mean(h), F.softplus(self.fc_var(h))

---

## 5. GCRLVAE.decode  (src/gcrl/models/gcrl_vae.py)

Add ct parameter:

    def decode(self, u: torch.Tensor,
               ct: Optional[torch.Tensor] = None) -> torch.Tensor:
        if self.cfg.ct_dim > 0 and ct is not None:
            u = torch.cat([u, ct], dim=-1)
        return F.leaky_relu(self.d2(F.leaky_relu(self.d1(u), 0.2)), 0.2)

---

## 6. GCRLVAE.forward  (src/gcrl/models/gcrl_vae.py)

Add ct parameter and pass it to encode/decode:

    def forward(self, x, c=None, c2=None, *, num_interv=1, temp=1.0,
                ct: Optional[torch.Tensor] = None):
        mu, var = self.encode(x, ct)
        z = self.reparameterize(mu, var)
        ...
        y_hat   = self.decode(self._dag_transform(...), ct)
        x_recon = self.decode(self._dag_transform(..., num_interv=0), ct)
        return y_hat, x_recon, mu, var, self.G

---

## 7. GCRLVAE.encode_batch / decode_batch / predict  (src/gcrl/models/gcrl_vae.py)

encode_batch and decode_batch currently ignore extra **_ kwargs. Make them accept ct:

    def encode_batch(self, X, ct=None):
        ...
        ct_t = self._to_tensor(ct) if ct is not None else None
        mu, var = self.encode(self._to_tensor(X), ct_t)
        ...

    def decode_batch(self, z, ct=None):
        ...
        ct_t = self._to_tensor(ct) if ct is not None else None
        return self.decode(self._to_tensor(z), ct_t).cpu().numpy()

In predict(), build the cell-type one-hot for each group and pass to encode/decode:

    ct_t = self._ct_one_hot(str(cell_type), n_pred)     # or None if ct_dim=0
    enc  = self.encode_batch(X_ctrl_in, ct=ct_t.cpu().numpy() if ct_t is not None else None)
    ...
    X_pred = self.decode_batch(u.cpu().numpy(), ct=ct_t.cpu().numpy() if ct_t is not None else None)

## 9. _PairedDataset  (src/gcrl/training/train_gcrl_vae.py)

Add cell-type integer codes so the dataloader can build one-hot vectors per batch:

    def __init__(self, ..., ct_codes: Optional[np.ndarray] = None, n_cell_types: int = 0):
        ...
        if ct_codes is not None and n_cell_types > 0:
            self.ct_codes     = torch.from_numpy(ct_codes.astype(np.int64))
            self.n_cell_types = n_cell_types
        else:
            self.ct_codes     = None
            self.n_cell_types = 0

    def __getitem__(self, i):
        ct = (F.one_hot(self.ct_codes[i], self.n_cell_types).float()
              if self.ct_codes is not None else torch.empty(0))
        return self.X_ctrl_enc[i], self.X_interv[i], self.C[i], self.X_ctrl[i], self.eig[i], ct

Note: ct_codes here are for the SAMPLED CONTROL cells (same as X_ctrl_enc[i]).

## 10. InterventionBatchSampler  (src/gcrl/training/train_gcrl_vae.py)

No changes needed. Batching is still by intervention condition. Cell-type information
is carried per-item in the dataset.

## 11. train_gcrl_vae  (src/gcrl/training/train_gcrl_vae.py)

In the setup block, derive ct_dim and build ct_codes for control cells:

    import pandas as pd
    ct_codes_ctrl, ct_uniques = pd.factorize(
        adata_ctrl.obs[_ct_key].astype(str).values, sort=True
    )
    ct_dim = len(ct_uniques) if len(ct_uniques) > 1 else 0
    # If only one cell type, no conditioning needed (ct_dim=0)

Pass ct_codes_ctrl and n_cell_types=ct_dim to _PairedDataset.

In GCRLVAEConfig, set ct_dim=ct_dim.

After model construction, call:
    if ct_dim > 0:
        model.set_ct_index(list(ct_uniques))

In the training loop, unpack ct from the batch and pass to model.forward:
    x_ctrl_enc, y_interv, c_bin, x_ctrl_all, eig, ct_batch = batch_items
    ct = ct_batch.to(device) if ct_dim > 0 else None
    y_hat, x_recon, mu, var, G = model(x_ctrl_enc, c_bin, c_bin,
                                        num_interv=1, temp=temp, ct=ct)

## 12. Batch sampler interaction

Currently batches mix cells from different cell types within the same intervention
condition. With conditioning this is fine — the model handles cell-type differences via
the one-hot ct vector. If you want homogeneous cell-type batches (e.g. to simplify
eigengene computation), the sampler key can be changed from intervention label to
(cell_type, intervention) tuple:

    group_key = f"{cell_type}__{intervention}"

This is a minor change to InterventionBatchSampler.__init__.
