1. BIOLOGICAL CONTEXT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Data:       Norman 2019 Perturb-seq, K562 cell line (erythroid-like cancer).
Goal:       Identify communities of transcription factors (TFs) that
            co-regulate similar target genes, using a gene regulatory
            network (GRN) inferred from the perturbation data.
Caveat:     K562 is myeloid/erythroid lineage → GO enrichment always
            returns myeloid differentiation terms at the top unless
            explicitly filtered out (see Section 5).

3. KEY DESIGN CHOICES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Clustering layer:
  • ONLY g_reg (regulatory layer). g_cot (co-target layer) is built but
    NOT used for community detection in this notebook ("no_g_cot").

Edge thresholding:
  • top_percentage=0.5 in control params (adjust based on GMM diagnostics).
  • The cumulative-weight Δ<5% criterion and GMM crossing both help choose
    a principled threshold. The left-tail GMM cutoff is a softer alternative.

Network layout:
  • layout="community": supervised spring layout.
    TF communities placed on a circle; within-community spring layout.
    Parameters: k=0.7, seed=42.
  • hide_isolates=True (singleton TFs excluded from plots).
  • Labels drawn with white bounding boxes via ax.annotate for readability.

FDR correction:
  • Per-cluster BH correction (NOT across all clusters pooled).
    This is the default in run_ora_for_clusters and must not be changed.

4. ORA UNIVERSE — CRITICAL DETAIL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
run_ora_for_clusters() supports three universe_mode values:

  'grn'    — genes in current_grn ∩ GO library.
             Problem: after intersection with GO library, only ~300–500
             genes remain (many GRN targets lack GO annotation).
             Cluster gene-sets cover 50–80% of this universe →
             hypergeometric test is underpowered → BH kills all results.

  'go'     — all genes in the GO library (~11,000).
             Problem: inflates enrichment of K562 housekeeping terms
             (myeloid differentiation, erythroid development) because
             cluster genes are a very small fraction of a huge universe.

  'custom' — recommended. Pass hvg_names (5,000 HVGs from AnnData).
             Balanced: larger than 'grn', smaller than 'go',
             tied to genes actually measured in this experiment.
             hvg_names is loaded in the control parameters cell.

Current setting: universe_mode='custom', custom_universe=hvg_names

5. BACKGROUND TERM FILTERING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
filter_cluster_specific_terms(ora_results, fdr_threshold=0.05,
                               max_cluster_fraction=0.5)

Flags GO terms significant (FDR<0.05) in >50% of clusters as
background_term=True. These are K562 cell-line artefacts, not
cluster-specific biology. Filtered_ora_results excludes them.

Typical flagged terms in this dataset:
  • Myeloid Cell Development (GO:0061515)
  • Myeloid Leukocyte Differentiation (GO:0002573)
  • Related erythroid/haematopoietic terms
