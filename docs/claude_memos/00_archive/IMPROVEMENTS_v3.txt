# gCRL-VAE Training Improvements - Version 3

This document describes the three major improvements made to the gCRL-VAE training system.

## Summary of Improvements

1. **GPU Detection and Confirmation** - Automatic GPU detection with detailed information display
2. **Training Progress Bars** - Real-time progress tracking with tqdm for epochs and batches
3. **Intelligent Auto-Configuration** - Dataset-aware parameter selection system

---

## 1. GPU Detection and Confirmation

### What was added:
- Automatic detection of GPU availability at training start
- Detailed GPU information display including:
  - GPU device name
  - GPU memory capacity
  - CUDA version
  - Clear warnings when training on CPU

### Location:
- File: `src/gcrl/training/train_gcrl_vae.py`
- Function: `train_gcrl_vae()` (at start)

### Example Output:
```
======================================================================
gCRL-VAE Training Configuration
======================================================================
✓ GPU ENABLED: NVIDIA GeForce RTX 3090
  - GPU Memory: 24.00 GB
  - CUDA Version: 11.8
Device: cuda
======================================================================
```

Or when no GPU is available:
```
======================================================================
gCRL-VAE Training Configuration
======================================================================
⚠ WARNING: Training on CPU (no GPU detected)
  Training will be significantly slower without GPU acceleration
Device: cpu
======================================================================
```

### Benefits:
- Immediate confirmation that GPU is being used
- Easy debugging when GPU is not detected
- No silent CPU training that might go unnoticed

## 2. Training Progress Bars

### What was added:
- Dual-level progress tracking using tqdm:
  - Epoch-level progress bar (outer loop)
  - Batch-level progress bar (inner loop)
- Real-time loss display in progress bars
- Key metrics shown: total loss, reconstruction loss, MMD loss, alignment loss

### Location:
- File: `src/gcrl/training/train_gcrl_vae.py`
- Modified: Training loop in `train_gcrl_vae()`

### Example Output:
```
Training gCRL-VAE:  45%|████████████▌            | 45/100 [15:23<18:52, 20.6s/epoch]
  Epoch 45/100: 100%|██████████████████| 35/35 [00:18<00:00,  1.89batch/s]
  loss=0.3421, rec=0.2134, mmd=0.0456, mcc=-0.8234
```

### Features:
- Estimated time remaining for epochs
- Current epoch and total epochs clearly displayed
- Loss values updated in real-time
- Batch progress within each epoch
- Clean, non-cluttering display (batch bar doesn't persist)

### Benefits:
- Know exactly how long training will take
- Monitor convergence in real-time
- Identify stuck or slow batches
- Professional appearance

### Fallback:
- If tqdm is not installed, training continues without progress bars
- Graceful degradation ensures compatibility

## 3. Intelligent Auto-Configuration

### What was added:
- New function: `analyze_dataset_and_suggest_config()`
- Comprehensive dataset analysis
- Intelligent parameter selection based on:
  - Dataset size (number of control cells)
  - Cell types (number and distribution)
  - Interventions (count and diversity)
  - Latent structure (number of communities)
  - TF/gene counts

### Location:
- File: `src/gcrl/training/train_gcrl_vae.py`
- Function: `analyze_dataset_and_suggest_config(adata, outdir, verbose=True)`

### Parameters Automatically Tuned:

#### 1. Batch Size
Based on: Minimum controls per cell type
```python
< 100 controls/CT:    batch_size = 64
< 500 controls/CT:    batch_size = 128
< 2000 controls/CT:   batch_size = 256
≥ 2000 controls/CT:   batch_size = 512
```

#### 2. Epochs
Based on: Total control cells
```python
< 1,000 cells:    epochs = 100
< 5,000 cells:    epochs = 75
< 20,000 cells:   epochs = 50
≥ 20,000 cells:   epochs = 40
```

#### 3. Learning Rate
Based on: Batch size
```python
batch_size ≤ 128:    lr = 1e-3
batch_size > 128:    lr = 2e-3
```

#### 4. Beta KLD Max (KL Divergence Weight)
Based on: Dataset size and latent dimensions
```python
< 2,000 cells OR > 10 latent dims:  beta_kld_max = 2e-2  (strong regularization)
< 10,000 cells:                     beta_kld_max = 1e-2  (medium)
≥ 10,000 cells:                     beta_kld_max = 5e-3  (mild)
```

#### 5. Lambda MCC (Alignment Weight)
Based on: Number of TF communities
```python
≤ 3 communities:    lambda_mcc = 1.0
≤ 6 communities:    lambda_mcc = 0.75
> 6 communities:    lambda_mcc = 0.5
```

#### 6. Lambda Sparse (Sparsity Weight)
Based on: Latent dimensions
```python
≤ 5 latent dims:     lambda_sparse = 0.01
≤ 10 latent dims:    lambda_sparse = 0.02
> 10 latent dims:    lambda_sparse = 0.05
```

#### 7. Alpha MMD Max (MMD Loss Weight)
Based on: Number of interventions
```python
< 10 interventions:   alpha_mmd_max = 1.5  (strong matching)
< 30 interventions:   alpha_mmd_max = 1.0  (standard)
≥ 30 interventions:   alpha_mmd_max = 0.75 (reduce weight)
```

#### 8. Lambda Centroid (Centroid Loss Weight)
Based on: Number of interventions
```python
< 20 interventions:   lambda_centroid = 0.1
≥ 20 interventions:   lambda_centroid = 0.05
```

### Usage in Notebook:

#### Option 1: Auto-configuration (Recommended)
```python
from gcrl.training.train_gcrl_vae import analyze_dataset_and_suggest_config

# Analyze and get suggested config
cfg = analyze_dataset_and_suggest_config(
    adata,
    outdir="results/VAE",
    verbose=True
)

# Optional: Override specific parameters
cfg.epochs = 150  # If you want more epochs
cfg.lambda_mcc = 0.5  # If you want different alignment weight
```

#### Option 2: Manual configuration
```python
from gcrl.training.train_gcrl_vae import VAEConfig

cfg = VAEConfig()
cfg.outdir = "results/VAE"
cfg.epochs = 100
cfg.batch_size = 256
# ... set all parameters manually
```

### Benefits:
- **Save time**: No need to manually tune parameters for each dataset
- **Better defaults**: Parameters adapt to dataset characteristics
- **Educational**: Analysis output explains why each parameter was chosen
- **Reproducible**: Verbose output documents the configuration reasoning
- **Flexible**: Can still override any parameter after auto-configuration
- **Prevents common mistakes**:
  - Batch size too large for small datasets
  - Too few epochs for small datasets
  - Regularization too weak/strong for dataset size
  - Mismatched parameters (e.g., large batch + small lr)


- All existing code continues to work without changes
- Auto-configuration is opt-in via the new function
- Progress bars work automatically (or silently degrade if tqdm missing)
- GPU detection runs automatically (no breaking changes)

## Design Rationale

### Why these specific parameter rules?

1. **Batch size vs controls/cell-type**:
   - Small batches when few controls prevent overfitting to single cell type
   - Large batches improve stability and training speed on large datasets
   - Cell-type batching means batch size should be < min(controls per CT)

2. **Epochs vs dataset size**:
   - Small datasets need more passes through data to converge
   - Large datasets converge faster per epoch
   - Prevents unnecessary computation on large datasets

3. **Learning rate vs batch size**:
   - Smaller batches have noisier gradients → need smaller lr
   - Larger batches have stabler gradients → can use larger lr

4. **Beta KLD vs dataset size**:
   - Small datasets overfit easily → need stronger regularization
   - Large datasets benefit from weaker regularization
   - High-dimensional latent spaces need stronger regularization

5. **Lambda MCC vs communities**:
   - Fewer communities → each alignment matters more → higher weight
   - Many communities → alignment distributed across many terms → can reduce weight

6. **Lambda Sparse vs latent dims**:
   - Higher dimensional spaces → more edges to regularize → need higher weight
   - Prevents over-parameterization of causal matrix G

7. **Alpha MMD vs interventions**:
   - Few interventions → each MMD computation very important → high weight
   - Many interventions → averaging across many → can reduce weight
   - Prevents over-emphasizing single intervention types

8. **Lambda Centroid vs interventions**:
   - Similar logic to MMD weight
   - Centroid loss complements MMD for mean shift effects

## Testing Recommendations

### Test on different dataset sizes:
- Small: < 1,000 control cells
- Medium: 1,000 - 10,000 control cells
- Large: > 10,000 control cells

### Test on different complexities:
- Few cell types (1-2)
- Many cell types (5+)
- Few interventions (< 10)
- Many interventions (30+)
- Low-dimensional latent (< 5)
- High-dimensional latent (> 10)

### Verify:
1. GPU detection works on both GPU and CPU systems
2. Progress bars display correctly in notebooks and terminals
3. Auto-configuration produces reasonable parameters
4. Training completes successfully with suggested parameters
5. Model performance comparable or better than manual tuning
