# Bug Fixes for 2_Norman_VAE.ipynb

## Issue Reported
AttributeError when accessing model attributes directly:
```python
model.z_dim  # ❌ FAILED - AttributeError: 'GCRLVAE' object has no attribute 'z_dim'
```

## Root Cause
The GCRLVAE model stores its configuration in `self.cfg` (a GCRLVAEConfig object), not as direct attributes on the model instance.

From gcrl_vae.py:
```python
class GCRLVAE(nn.Module):
    def __init__(self, cfg: GCRLVAEConfig, device: Optional[torch.device] = None):
        super().__init__()
        self.cfg = cfg  # <-- Config stored here
        # ...
```

## Fix Applied

Changed cell `07396a4c-70a2-43d8-94b9-c6fb96b70e6a` to access attributes via `model.cfg`:

### Before (Broken):
```python
print(f"Latent dimension (z_dim): {model.z_dim}")
print(f"Intervention dimension (c_dim): {model.c_dim}")
print(f"Input dimension: {model.input_dim} (TFs only)")
print(f"Output dimension: {model.output_dim} (all genes)")
if hasattr(model, 'ct_dim'):
    print(f"Cell-type dimension: {model.ct_dim}")
```

### After (Fixed):
```python
print(f"Latent dimension (z_dim): {model.cfg.z_dim}")
print(f"Intervention dimension (c_dim): {model.cfg.c_dim}")
print(f"Input dimension: {model.cfg.input_dim} (TFs only)")
print(f"Output dimension: {model.cfg.output_dim} (all genes)")
if hasattr(model.cfg, 'ct_dim'):
    print(f"Cell-type dimension: {model.cfg.ct_dim}")
print(f"\nDAG causal matrix G shape: {model.G.shape}")
print(f"Intervention mapping: {model.cfg.intervention_mapping}")
print(f"Using polynomial decoder: {model.cfg.use_polynomial_decoder}")
```

## Related Model Attributes

### Direct Access (on model instance):
- `model.G` - Causal matrix (nn.Parameter)
- `model.encoder` - Encoder network
- `model.fc_mu` - Mean projection layer
- `model.fc_var` - Variance projection layer
- `model.decoder_linear` - Linear decoder layer
- `model.decoder_quadratic` - Quadratic decoder layer
- `model.c_shift` - Intervention shift parameters
- `model.tf_index` - TF name to index mapping
- `model.tf_to_latent` - TF to latent dimension mapping
- `model.ct_index` - Cell type to index mapping

### Via Config (model.cfg):
- `model.cfg.input_dim` - Number of input features (TFs)
- `model.cfg.z_dim` - Latent dimension (p+1)
- `model.cfg.c_dim` - Number of intervention TFs
- `model.cfg.output_dim` - Number of output features (all genes)
- `model.cfg.ct_dim` - Number of cell types
- `model.cfg.intervention_mapping` - "hard" or "soft"
- `model.cfg.use_polynomial_decoder` - Boolean
- `model.cfg.hidden_enc` - Encoder hidden dimensions tuple
