# Final Fixes to gCRL-VAE (Post Initial Implementation)

## Date: 2025-01-XX

This document describes the critical fixes applied after the initial v2 implementation.

---

## Fix 1: MMD on Original Expression (CRITICAL)

### Problem
MMD was comparing simulated expression against **reconstructed** real intervened cells:
```python
# WRONG: Passing real cells through encoder-decoder
X_real_tf = X_real_all[:, tf_idx]
mu_real, var_real = model.encode(X_real_tf, ct=ct_real_vec)
z_real = model.reparameterize(mu_real, var_real)
x_real_rec = model.decode(z_real, ct=ct_real_vec)

loss_mmd = MMD(x_sim, x_real_rec)  # Adds unnecessary reconstruction noise
```

**Why this is wrong:**
- Real intervened cells ARE the ground truth
- Passing them through encoder-decoder adds noise
- We're comparing "simulated" vs "reconstructed real" instead of "simulated" vs "actual real"

### Solution
Compare directly against original expression:
```python
# CORRECT: Use original real expression
loss_mmd = MMD(x_sim, X_real_all)  # Direct comparison
```

**Why this is correct:**
- Real intervened cells are the actual biological outcome
- No reconstruction noise
- Direct comparison: model prediction vs. ground truth
- Both in same space (z-scored expression)

### Impact
- More accurate intervention learning
- Cleaner gradient signal
- Removes unnecessary computation

## Fix 2: Centroid Loss Scheduling

### Problem
Centroid loss had constant weight while MMD had scheduled weight:
```python
# INCONSISTENT: Different schedules
loss = loss + alpha_mmd * loss_mmd          # Scheduled (0 → 1.0)
loss = loss + lambda_centroid * loss_centroid  # Constant (0.1)
```

This causes:
- Centroid dominates early in training (before MMD ramps up)
- Inconsistent intervention learning dynamics
- Harder to tune hyperparameters

### Solution
Apply same schedule to both:
```python
# CONSISTENT: Same schedule for both
loss = loss + alpha_mmd * loss_mmd
loss = loss + alpha_mmd * lambda_centroid * loss_centroid  # Now scheduled!
```

**Schedule:**
- Epochs 1-5: Both at 0 (no intervention loss)
- Epochs 6+: Both ramp linearly to max (MMD: 1.0, Centroid: 0.1)

### Benefits
- Consistent intervention learning
- Both components activate together
- Simpler hyperparameter tuning
- Prevents centroid from dominating early

## Fix 3: Adjusted Loss Weights

### Problem
Original weights were imbalanced:

| Component | Weight | Max Value | Relative Strength |
|-----------|--------|-----------|-------------------|
| L_rec | 1.0 | 1.0 | Strong |
| L_KL | 0 → 1e-3 | 0.001 | **Very Weak** |
| L_mcc | 1.0 | 1.0 | Strong |
| L_sparse | 0.01 | 0.01 | Weak |
| L_MMD | 0 → 1.0 | 1.0 | Strong |
| L_centroid | 0.1 | 0.1 | Medium |

**Issue:** KL weight is 1000× smaller than reconstruction, may not regularize enough.

### Solution
Increased KL weight by 10×:
```python
# Before
beta_kld_max = 1e-3

# After
beta_kld_max = 1e-2  # Stronger regularization
```

**New relative strengths:**

| Component | Weight | Max Value | Relative Strength |
|-----------|--------|-----------|-------------------|
| L_rec | 1.0 | 1.0 | Strong |
| L_KL | 0 → 1e-2 | **0.01** | **Weak-Medium** |
| L_mcc | 1.0 | 1.0 | Strong |
| L_sparse | 0.01 | 0.01 | Weak |
| L_MMD | 0 → 1.0 | 1.0 | Strong |
| L_centroid | (scheduled) | 0.1 | Medium |

### Rationale
- KL prevents latent collapse
- Still weak enough not to destroy information
- Can be tuned based on dataset:
  - Small datasets: increase to 0.1
  - Large datasets: decrease to 1e-3

## Final Loss Function

```python
# Complete loss with all fixes
L_total = 1.0 × L_rec                          # Reconstruction
        + β(t) × L_KL                          # β: 0 → 0.01 (epochs 10+)
        + 1.0 × L_mcc                          # Eigengene alignment
        + 0.01 × L_sparse                      # Sparsity
        + α(t) × L_MMD                         # α: 0 → 1.0 (epochs 5+)
        + α(t) × 0.1 × L_centroid              # Same α schedule
```

**Where:**
- `L_MMD = MMD(x_sim, X_real_all)` - Direct comparison with original
- `L_centroid = MSE(mean(x_sim), mean(X_real_all))` - Mean shift matching
- Both use `α(t)` schedule for consistency

## Hyperparameter Recommendations

### Default (Recommended):
```python
VAEConfig(
    alpha_rec=1.0,
    lambda_mcc=1.0,
    lambda_sparse=0.01,
    lambda_centroid=0.1,      # Uses MMD schedule
    beta_kld_max=1e-2,        # Increased from 1e-3
    alpha_mmd_max=1.0,
)
```

### For Small Datasets (<1000 cells):
```python
beta_kld_max=0.1,       # Stronger regularization
lambda_sparse=0.1,      # Sparser graphs
lambda_centroid=0.2,    # Stronger mean matching
```

### For Large Datasets (>10000 cells):
```python
beta_kld_max=1e-3,      # Weaker regularization
lambda_centroid=0.05,   # Let MMD dominate
```

### To Disable Centroid Loss:
```python
lambda_centroid=0.0     # MMD only
```

## Theoretical Justification

### Why Original Expression for MMD?

**Intervention Learning Goal:**
```
Learn: Control + Intervention → Predicted Outcome
Validate: Predicted Outcome ≈ Real Outcome
```

**Correct Approach:**
```python
Predicted = Model(Control, Intervention)  # Simulated
Real = Actual_Intervened_Cells           # Ground truth
Loss = Distance(Predicted, Real)
```

**Incorrect Approach:**
```python
Predicted = Model(Control, Intervention)
Real_Reconstructed = Model.decode(Model.encode(Real))  # Adds noise!
Loss = Distance(Predicted, Real_Reconstructed)  # Comparing to noisy version
```

**Why this matters:**
- Model learns to match its own noise patterns instead of biology
- Gradients become less informative
- Training is less efficient

## Expected Training Behavior

With these fixes, expect:

1. **Epochs 1-5**:
   - Only L_rec, L_KL (ramping), L_mcc, L_sparse
   - Model learns basic reconstruction + alignment

2. **Epochs 6-10**:
   - MMD + Centroid start ramping (both together)
   - Model begins learning interventions

3. **Epochs 11+**:
   - KL fully ramped (0.01)
   - MMD + Centroid fully ramped (1.0, 0.1)
   - All components active

**Loss component magnitudes (typical):**
- L_rec: 0.1 - 1.0
- L_KL: 0.001 - 0.01 (after ramping)
- L_mcc: -0.5 to -0.1 (negative, want ≈ 0)
- L_sparse: 0.01 - 0.1
- L_MMD: 0.1 - 1.0 (after ramping)
- L_centroid: 0.01 - 0.1 (after ramping)
