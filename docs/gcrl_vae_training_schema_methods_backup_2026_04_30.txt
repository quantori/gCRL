### gCRL-VAE: Causal Variational Autoencoder with Gene Regulatory Network Priors

We developed gCRL-VAE (Gene Community Representation Learning - Variational Autoencoder), a causal deep learning framework that learns biologically interpretable latent representations aligned with gene regulatory network (GRN) structure while enabling accurate simulation of transcription factor (TF) perturbations.

#### Architecture and Causal Structure

The gCRL-VAE architecture extends the standard VAE framework with a directed acyclic graph (DAG) transform layer that enforces causal structure. A critical design innovation is the use of **TF-only input** and **all-gene output**: the encoder accepts only transcription factor (TF) expression **x**_TF ∈ ℝ^n_TF (where n_TF is the number of TFs), while the decoder reconstructs expression for all genes **x̂** ∈ ℝ^d (where d = n_TF + n_TG is the total number of TFs and target genes). This architecture forces the model to learn explicit TF → gene regulatory mappings, reflecting the biological causality where TFs drive target gene expression.

The TF-only input **x**_TF is encoded via a simplified MLP with architecture [n_TF → 128 → 2(*p*+1)] to distributional parameters (μ, σ²), from which an observational latent **z** ~ N(μ, σ²) is sampled via the reparameterization trick. To model interventions, we apply a DAG transform: **u** = (I − G)⁻¹(**z** + δ), where G ∈ ℝ^((*p*+1)×(*p*+1)) is an upper-triangular learnable causal matrix, and δ represents the intervention shift routed to specific latent dimensions based on TF community assignment. The intervened latent **u** is then decoded via a **polynomial decoder** (linear + quadratic terms) with no hidden layers: **x̂** = W_1**u** + W_2(**u** ⊗ **u**), where ⊗ denotes the outer product flattened to a vector. This polynomial structure ensures mathematical interpretability and supports theoretical identifiability guarantees, avoiding the arbitrary non-linearities of standard MLPs.

The latent space has dimensionality *p* + 1, where *p* is the number of unique TF communities observed in perturbation experiments, plus one global dimension capturing TF-wide variation. This community-aware structure ensures biological interpretability, with each latent dimension corresponding to a regulatory module. The upper-triangular constraint on G ensures acyclicity by defining an ordering over latent variables, preventing feedback loops in the causal graph.

#### Data Preparation and Preprocessing

Input data consists of single-cell RNA-seq experiments stored as AnnData objects with normalized and log₁₊ₓ-transformed expression values (X → log(1 + X/size_factor)). Gene metadata includes `kind` (TF or target gene) and `community` (TF community assignment derived from GRN modularity analysis or prior biological knowledge). Cell metadata specifies `cell_type`, `intervention` (perturbed TF names, combinatorial perturbations separated by delimiters [+, ;, or ,], or control markers ["control", "unperturbed", "na"]), and `set` (training or test split).

Prior to training, expression values undergo per-cell-type z-score normalization using **training control cells only** to prevent information leakage. For each cell type c, we compute gene-wise means μ_c,g and standard deviations σ_c,g across unperturbed cells. To prevent numerical instability from near-zero variance genes, we apply shrinkage regularization: σ̂_c,g = √(σ²_c,g + ε) with ε = 0.1. The z-scored expression is then computed as z_c,i,g = (x_c,i,g − μ_c,g) / σ̂_c,g and clipped to [−6, 6] to mitigate outlier effects. These frozen statistics are applied to all cells (both training and test, controls and perturbations), ensuring that test-time predictions use only training-derived normalization parameters. For cell types absent from training controls, we apply global fallback statistics computed across all training control cells. The original log₁₊ₓ-transformed values are preserved in a separate data layer for reference.

Intervened TFs are parsed from the `intervention` column (e.g., "GATA1+TAL1" → ["GATA1", "TAL1"]), validated against gene annotations (ensuring presence in expression matrix and annotation as `kind` = "TF"), and mapped to latent dimensions via their community assignments. Hard routing establishes a deterministic mapping from each intervened TF to its corresponding community-specific latent dimension k, where k = comm2latent[community[TF]]. This mapping is stored and used consistently during training and inference.

#### Training Strategy: Controls-Only Reconstruction

A critical design choice distinguishes gCRL-VAE from prior causal VAE methods: training batches contain **only unperturbed control cells**, stratified by cell type. We implement a custom batch sampler that ensures each mini-batch (default size: 256 cells) contains cells exclusively from a single cell type, cycling through cell types randomly without replacement until all cells have been sampled per epoch. This single-cell-type batching strategy serves two purposes: (1) it ensures homogeneous eigengene computation within batches, as regulatory module activity may vary across cell types, and (2) it prevents confounding from cell-type effects during alignment loss calculation.

Control cells are used for VAE reconstruction, KL regularization, and eigengene alignment losses. Importantly, **perturbed cells are never passed through the encoder-decoder during reconstruction**; instead, they serve exclusively as ground truth for validating simulated interventions via the MMD loss (detailed below). Perturbation data is organized into intervention-specific groups: for each unique perturbation condition (e.g., "GATA1", "TAL1", "GATA1+TAL1"), we maintain a separate pool containing the expression profiles and cell-type labels of all cells subjected to that perturbation.

This controls-only reconstruction strategy ensures the model learns the baseline unperturbed expression manifold without contamination from intervention-induced transcriptional changes, while intervention effects are learned through distributional comparison between simulated and real perturbations.

#### Cell-Type Conditioning

When multiple cell types are present (*n* > 1), the model operates as a conditional VAE with cell-type conditioning dimension *ct_dim* = *n*. One-hot encoded cell-type vectors are concatenated to both encoder inputs and decoder latents, enabling the model to capture cell-type-specific expression patterns and intervention responses.

#### Complete Loss Function

The training objective comprises seven complementary terms with distinct biological motivations:

**L**total = α_rec **L**rec + β(*t*) **L**KL + λ_mcc **L**align + λ_sparse **L**sparse + λ_dag **L**dag + α(*t*) **L**MMD + α(*t*) × λ_centroid **L**centroid

**Reconstruction loss** (**L**rec) measures mean squared error between TF-only input and all-gene reconstructed expression for control cells, weighted by α_rec = 1.0. Given TF expression **x**_TF as input, the model predicts all-gene expression **x̂**, and reconstruction error is computed against the full expression profile **x**. This term learns the baseline gene expression manifold and the TF → gene mappings.

**KL divergence loss** (**L**KL = −0.5 × mean(1 + log σ² − μ² − σ²)) regularizes the latent space toward an isotropic Gaussian prior N(0, I). Crucially, the encoder outputs variance σ² (not log-variance), which is converted for the KL formula. The weight β(*t*) follows a scheduled ramp: β = 0 for epochs 1–10, then linearly increases to β_max = 10⁻², allowing the model to learn meaningful structure before strong regularization. The increased weight (10⁻² vs. previous 10⁻³) provides stronger regularization to prevent latent collapse while remaining weak enough to preserve information.

**Eigengene alignment loss** (**L**align) enforces correspondence between latent dimensions and TF community structure. For each training batch, we construct an eigengene matrix **E** (size B × (*p* + 1)) where columns 0..(*p* − 1) contain the first principal component (PC1) of TF expression within each community, and column *p* is the PC1 of all TFs (global eigengene). PC1 scores are computed using power iteration: given TF expression matrix X_TF for TFs in community c, we center the data (X_c = X_TF − mean(X_TF)) and iteratively compute v^(t+1) = X_c^T X_c v^(t) / ||X_c^T X_c v^(t)|| for 12 iterations, then project: E_c = z-score(X_c v^(12)). This on-the-fly computation ensures eigengenes reflect the current batch composition, though precomputed eigengenes can be used if available in adata.obsm["X_comm_eig"].

The partial maximum correlation coefficient (partial-MCC) loss computes partial correlations between encoder means **MU**[:, 0:*p*+1] and **E**, controlling for inter-dimension dependencies. Given two matrices E and MU of size B × (*p*+1), partial-MCC computes the partial correlation r_i between E[:, i] and MU[:, i] while conditioning on all other dimensions. The loss is the negative mean partial correlation: **L**align = −(1/(*p*+1)) Σ_i r_i. Minimizing this loss (λ_mcc = 1.0) maximizes one-to-one alignment between each eigengene and its corresponding latent dimension, while accounting for biological crosstalk between regulatory modules that may introduce correlations between dimensions.

**Sparsity loss** (**L**sparse = ||G||₁) encourages a sparse causal graph by applying L1 regularization to the full G matrix excluding the diagonal (λ_sparse = 0.001). **UPDATED (2025-11-13)**: The L1 penalty was reduced 10-fold (from 0.01 to 0.001) to act as a gentle tie-breaker rather than a strong constraint, and is applied to the full matrix rather than just the upper triangle. The penalty is only active from epoch 10 onwards (sparse_start_epoch = 10), allowing early learning freedom. This constraint promotes interpretable causal structures where only a subset of latent dimensions influence downstream dimensions, consistent with modular regulatory architecture.

**NOTEARS DAG acyclicity constraint** (**L**dag = tr(exp(G ⊙ G)) - d) enforces that the learned causal matrix G forms a directed acyclic graph (DAG) using the smooth differentiable constraint from Zheng et al. (2018). Here, G_offdiag = G with diagonal zeroed to remove self-loops, A = G_offdiag ⊙ G_offdiag (elementwise square), exp_A = matrix_exp(A), and d is the latent dimension. The constraint h(G) = tr(exp_A) - d equals zero if and only if G is acyclic. This allows gradient-based optimization to discover the optimal DAG structure from data without imposing a fixed causal ordering (e.g., upper-triangular constraint). The NOTEARS penalty uses a strong weight (λ_dag = 1.0) and is active from epoch 10 onwards (dag_start_epoch = 10). This methodology enables the model to learn flexible causal relationships while ensuring the graph remains acyclic, a critical requirement for valid causal inference. The matrix exponential computation is differentiable and compatible with backpropagation, though it can be expensive for large latent dimensions.

**Maximum Mean Discrepancy loss** (**L**MMD) validates that simulated interventions match real perturbation effects. At each training step, we sample a random intervention type from the available perturbation conditions and select min(batch_size/2, n_available) real perturbed cells from that condition. We then sample an equal number of control cells (potentially from matching cell types to minimize confounding) and simulate the same intervention by encoding the control TFs, applying the DAG transform with intervention vector **c** (a binary vector indicating which TFs are perturbed), decoding to obtain simulated expression **x̂**_sim, and comparing against the **original expression** of real perturbed cells **x**_real.

**Critically**, MMD is computed on **reconstructed gene expression space** (**x̂**_sim vs. **x**_real), NOT on DAG-transformed latents. This is a key design decision: real perturbed cells already represent the biological outcome of the intervention, so we compare the model's predicted intervention outcome directly against the ground truth expression without passing real cells through the encoder-decoder (which would add unnecessary reconstruction noise). The algorithm is:

1. Sample control cells and extract TF expression: **x**_ctrl,TF
2. Encode controls: (μ, σ²) = Encoder(**x**_ctrl,TF)
3. Sample latent: **z** ~ N(μ, σ²)
4. Apply intervention: **u**_sim = (I − G)⁻¹(**z** + δ)
5. Decode to all-gene expression: **x̂**_sim = Decoder(**u**_sim)
6. Sample real perturbed cells: **x**_real (original expression, all genes)
7. Compute MMD: **L**_MMD = MMD(**x̂**_sim, **x**_real)

The MMD kernel employs multi-scale Gaussian kernels with adaptive bandwidth selection. Given source and target distributions with combined sample matrix **X** ∈ ℝ^(2n × d), we compute pairwise squared Euclidean distances D_ij = ||x_i − x_j||². The bandwidth is set adaptively as σ² = Σ_ij D_ij / (4n² − 2n) unless manually specified. We then generate 5 kernel scales with bandwidths {σ²/4, σ²/2, σ², 2σ², 4σ²} (bandwidth multiplication factor = 2.0). For each scale s, we compute kernel matrix K^(s)_ij = exp(−D_ij / σ²_s). The final MMD² estimate is:

MMD²(**x̂**_sim, **x**_real) = (1/n²) Σ_ij K_ij^XX + (1/n²) Σ_ij K_ij^YY − (2/n²) Σ_ij K_ij^XY

where superscripts denote source-source, target-target, and source-target blocks of the kernel matrix, averaged across all 5 scales.

The weight α(*t*) follows a scheduled ramp: α = 0 for epochs 1–5, then linearly increases to α_max = 1.0, allowing the encoder and decoder to stabilize on reconstruction before enforcing intervention distribution matching.

**Centroid loss** (**L**centroid) is an optional sixth component that directly targets mean expression shift matching between simulated and real interventions. Given simulated expression **x̂**_sim and real perturbed expression **x**_real, we compute gene-wise means (centroids): **μ**_sim = mean(**x̂**_sim, axis=0) and **μ**_real = mean(**x**_real, axis=0), then minimize **L**centroid = MSE(**μ**_sim, **μ**_real). This loss explicitly enforces that the model reproduces the mean transcriptional shift induced by perturbations, complementing the full distributional matching of MMD. Importantly, centroid loss uses the **same schedule as MMD**: the effective weight is α(*t*) × λ_centroid, where α(*t*) ramps from 0 to 1.0 during epochs 6+ and λ_centroid = 0.1 is the constant weight factor. This consistent scheduling ensures both components activate together, preventing one from dominating early in training. Centroid loss can be disabled by setting λ_centroid = 0; it is most useful for datasets with strong mean shift effects where MMD alone may have high variance.

#### Optimization and Implementation

Model parameters are optimized using the Adam optimizer (learning rate lr = 2 × 10⁻³, β₁ = 0.9, β₂ = 0.999) with gradient norm clipping (max norm = 0.5) to prevent exploding gradients in the DAG inverse computation. The scheduled ramping of β (KL weight), α (MMD and centroid weights), and late-onset structural penalties enables staged learning:
- **Epochs 1–5**: Reconstruction (α_rec = 1.0) and eigengene alignment (λ_mcc = 1.0) dominate, establishing basic encoder-decoder correspondence and latent structure without structural constraints
- **Epochs 6–10**: MMD loss (α ramping to 1.0) and centroid loss (α × 0.1) are introduced together, beginning to match intervention distributions while KL remains zero; no DAG or sparsity penalties yet
- **Epochs 10+**: Sparsity (λ_sparse = 0.001) and DAG acyclicity (λ_dag = 1.0) penalties activate, guiding G toward sparse acyclic structure
- **Epochs 11+**: KL divergence (β ramping to 10⁻²) regularizes latent space, with all seven loss terms active

This curriculum prevents premature collapse to trivial solutions and ensures biologically meaningful structure emerges before strong regularization. The late onset of structural penalties (epoch 10) allows the model initial freedom to explore the causal structure space before constraining it to sparse acyclic graphs.

Training proceeds for 50–100 epochs depending on dataset size (50 epochs typical for datasets with <20,000 cells; 100+ for larger datasets). All seven loss components are logged per batch and aggregated per epoch, including scheduled weights β(*t*), α(*t*), and temperature τ(*t*) (for soft routing, if applicable), enabling detailed analysis of training dynamics. This includes tracking the NOTEARS DAG penalty (loss_dag) to monitor convergence toward acyclicity. Model checkpoints, preprocessing metadata (including frozen z-score parameters), and evaluation wiring (TF names, TF-to-latent mapping, community assignments) are saved in JSON format for downstream analysis and intervention prediction.

The implementation is performed in PyTorch with CUDA support for GPU acceleration. All computations use single-precision floating-point (float32) arithmetic. Random seeds are set for reproducibility across NumPy, PyTorch CPU, and PyTorch CUDA backends.

#### Validation and Intervention Simulation

Post-training, the model enables zero-shot prediction of unseen single and combinatorial TF perturbations. For a given intervention (specified as a set of TF names), we construct a binary intervention vector **c** ∈ {0,1}^c_dim where c_i = 1 if TF_i is perturbed and 0 otherwise. For inference:

1. Encode control cells: (μ, σ²) = Encoder(x_ctrl, ct)
2. Sample observational latent: z ~ N(μ, σ²) or use mean z = μ for deterministic prediction
3. Apply intervention shift: δ = bc ⊙ c_shift, where bc performs hard routing to map each intervened TF to its corresponding latent dimension, and c_shift is a learned per-TF intervention magnitude
4. Compute intervened latent: u = (I − G_offdiag)⁻¹(z + δ), where G_offdiag removes diagonal from G
5. Decode to predicted expression: x̂_interv = Decoder(u, ct)

Cell-type-specific predictions leverage the conditional VAE structure by providing the appropriate one-hot encoded cell-type vector **ct** during both encoding and decoding. The community-structured latent space ensures that interventions on related TFs (same community) produce correlated effects, as they map to the same latent dimension, while the learned DAG structure G (enforced via NOTEARS penalty rather than structural constraint) propagates intervention effects through discovered causal dependencies between latent variables.

For model evaluation, we assess: (1) **reconstruction accuracy** on held-out control cells (MSE, Pearson correlation per gene), (2) **intervention prediction accuracy** on held-out perturbed cells (comparing predicted vs. observed differential expression), (3) **latent alignment quality** (partial correlation between eigengenes and latent dimensions), (4) **causal graph interpretability** (sparsity of G, correspondence with known regulatory relationships), and (5) **generalization to combinatorial perturbations** (accuracy on double/triple knockouts not seen during training).

#### Comparison to Discrepancy-VAE

While gCRL-VAE builds on the discrepancy-VAE framework (Shen et al., 2021) for causal intervention modeling, it introduces key innovations tailored to single-cell transcriptomics with GRN priors: (1) **TF-only input with all-gene output** versus all-gene input/output, forcing explicit TF → gene mappings, (2) **polynomial decoder with quadratic terms** (linear + quadratic) extending discrepancy-VAE's two-layer linear decoder, enabling explicit modeling of interaction effects, (3) **controls-only reconstruction** versus all-cells reconstruction, ensuring clean baseline learning, (4) **cell-type stratified batching** versus standard batching, (5) **eigengene alignment loss** (partial-MCC) enforcing biological structure alignment with TF community eigengenes, (6) **community-based latent dimensions** (p+1 structure) versus unconstrained dimensions, (7) **cell-type conditioning** via one-hot concatenation for multi-cell-type datasets, (8) **MMD computed on original expression space** (simulated vs. real) without reconstructing real intervened cells, avoiding unnecessary noise, (9) **centroid loss with consistent scheduling** to directly target mean shifts, (10) **stronger KL regularization** (10⁻² vs. typical 10⁻³) for better latent structure, and (11) **NOTEARS DAG acyclicity constraint** enabling data-driven causal structure discovery rather than imposing fixed orderings (e.g., upper-triangular constraints). These modifications ensure the learned representations reflect underlying regulatory biology while maintaining intervention prediction accuracy.

#### Hyperparameter Selection and Model Configuration

The hyperparameter configuration was chosen based on the following considerations:
- **Batch size**: 256 cells (reduced to 128 for small datasets <5,000 cells)
- **Learning rate**: 2 × 10⁻³ (Adam optimizer defaults)
- **Epochs**: 50–100 (50 sufficient for <20,000 cells; 100+ for larger datasets)
- **Loss weights** (constant): α_rec = 1.0, λ_mcc = 1.0, λ_sparse = 0.001, λ_dag = 1.0, λ_centroid = 0.1
- **Loss weights** (scheduled max): β_kld_max = 10⁻², α_mmd_max = 1.0
- **MMD kernel**: 5 scales, bandwidth multiplier = 2.0, adaptive bandwidth
- **Preprocessing**: ε = 0.1 (shrinkage), clip range = [−6, 6]
- **Gradient clipping**: max norm = 0.5
- **Architecture**: Encoder [n_TF → 128 → 2(*p*+1)], Decoder polynomial [(*p*+1) → n_genes] (linear + quadratic)

For datasets with >5 cell types, we recommend increasing encoder hidden layer size to [n_TF → 256 → 2(*p*+1)] or adding a second layer [n_TF → 128 → 128 → 2(*p*+1)] to accommodate increased model capacity requirements. For datasets with weak perturbation effects, increasing α_mmd_max to 2.0 and starting MMD ramping at epoch 3 (instead of 5) can improve intervention learning. For datasets with strong mean shifts, increasing λ_centroid to 0.2 or 0.5 enhances centroid matching; conversely, set λ_centroid = 0 to disable if redundant with MMD.

#### Computational Requirements

Training time scales approximately linearly with dataset size. On an NVIDIA A100 GPU (40GB), typical training times are: ~10 minutes for 5,000 cells, ~30 minutes for 20,000 cells, and ~2 hours for 100,000 cells (all for 50 epochs). Peak GPU memory usage is approximately 4–8 GB for typical datasets. CPU-only training is supported but substantially slower (5–10× longer). Inference on trained models is fast: ~1 second per 10,000 cells for intervention prediction.

#### Summary

By integrating TF-only input architecture, polynomial decoder, control-based normalization, community-aware latent structure, eigengene alignment, and distributional intervention validation with original expression, gCRL-VAE achieves biologically interpretable causal representations that enable accurate simulation of TF perturbations. The scheduled multi-objective training procedure with six loss components balances reconstruction fidelity, latent regularization, biological alignment, causal sparsity, full distribution matching (MMD), and mean shift matching (centroid), producing models that generalize to unseen perturbations while preserving consistency with gene regulatory network topology. The TF → gene architecture and polynomial decoder provide theoretical interpretability advantages over standard VAEs, while the simplified encoder prevents overfitting on small sample sizes typical of single-cell perturbation screens. The framework is applicable to any single-cell perturbation dataset with annotated TF communities and supports both single and multi-cell-type experiments.
