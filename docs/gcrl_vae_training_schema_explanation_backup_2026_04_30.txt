## Overview

gCRL-VAE (Gene Community Representation Learning - Variational Autoencoder) is a causal
VAE that learns biologically interpretable latent representations aligned with gene
regulatory network (GRN) structure. The model is trained on control (unperturbed) cells
and learns to simulate transcription factor (TF) perturbations through a causal DAG
structure.

**Key Innovation**: Unlike standard VAEs, gCRL-VAE:
1. Uses controls-only reconstruction (no perturbed cells in reconstruction loss)
2. Learns causal structure via DAG-transformed latent space
3. Validates interventions against real perturbed cells using MMD loss
4. Aligns latent dimensions with TF community eigengenes
5. Supports cell-type conditioning for multi-cell-type datasets

## Architecture

```
INPUT: TF expression x_TF (n_TFs only)
   ↓
[ENCODER] → (μ, σ²)          # Encodes TFs to latent distribution parameters
   ↓
z ~ N(μ, σ²)                 # Sample observational latent (pre-intervention)
   ↓
[DAG TRANSFORM]              # Apply intervention via causal graph
   ↓
u = (I - G)^{-1} @ (z + δ)   # Intervened latent (post-intervention)
   ↓
[POLYNOMIAL DECODER] → x_rec # Reconstruct ALL gene expression (TF + TG)
```

### Notation:
- **x_TF**: Input TF expression only (z-scored, controls only during training)
- **z**: Observational latent (before intervention)
- **u** (or y_hat in code): Intervened latent (after DAG transform)
- **x_rec**: Reconstructed gene expression for ALL genes (TF + TG)
- **G**: Upper-triangular causal matrix (learnable parameter)
- **δ**: Intervention shift (routed to specific latent dimension)

### Key Innovation: TF-Only Input → All-Gene Output
- **Encoder input**: Only TF expression (input_dim = n_TFs)
- **Decoder output**: All genes (output_dim = n_genes = n_TFs + n_TGs)
- **Rationale**: TFs are the causal drivers; forcing model to learn TF→gene mappings

### Latent Space Structure:
- **Dimension**: z_dim = p + 1
  - Dimensions 0..p-1: One per intervened TF community
  - Dimension p: Global latent (captures TF-wide variation)
- **Routing**: Hard mapping from TFs to latent dimensions via community assignment

## Data Requirements

### Input AnnData Structure:

**adata.X**: Normalized, log1p-transformed gene expression
- Will be z-scored per cell type using training controls only
- Original values preserved in `adata.layers["X_log1p"]`

**adata.obs** (cell metadata):
- `set`: "training" or "test" (only training cells used for model fitting)
- `cell_type`: Categorical cell type labels (for stratified batching & conditioning)
- `intervention`: Intervention descriptor
  - Control cells: "control", "unperturbed", "na", "", or similar
  - Single interventions: "TF1", "GATA1", etc.
  - Combinatorial: "TF1+TF2" (separated by +, ;, or ,)

**adata.var** (gene metadata):
- `kind`: "TF" or "TG" (transcription factor or target gene)
- `community`: Community assignment for TFs (categorical)

**adata.obsm** (optional):
- `X_comm_eig`: Precomputed eigengenes (n_cells × (p+1))
  - Columns 0..p-1: Community eigengenes
  - Column p: Global eigengene
- If missing, computed on-the-fly during training

**adata.uns** (optional metadata for eigengenes):
- `X_comm_eig_comm_ids`: List of community IDs corresponding to columns
- `X_comm_eig_global_index`: Index of global eigengene column

## Training Data Separation

**CRITICAL DESIGN CHOICE**: Controls and interventions are used differently.

### Controls (Unperturbed Cells):
- **Usage**: VAE reconstruction, eigengene alignment, KL regularization
- **Batching**: Single cell-type batches for homogeneous eigengene computation
- **Purpose**: Learn the baseline (unperturbed) gene expression manifold

### Interventions (Perturbed Cells):
- **Usage**: Ground truth for MMD loss ONLY
- **NOT passed through encoder/decoder during reconstruction**
- **Purpose**: Validate that simulated interventions match real perturbations
- **Grouped by**: Intervention type for per-intervention MMD computation

### Data Flow:
```
Training Set (adata.obs["set"] == "training")
    ↓
    ├─→ Controls → VAE training batches → Reconstruction + KL + Alignment
    └─→ Interventions → Grouped by type → MMD loss computation
```

## Preprocessing Pipeline

### 1. Per-Cell-Type Z-Scoring (Controls-Only):

**Goal**: Standardize expression while preventing data leakage from perturbations.

**Procedure**:
1. Identify training controls: `set == "training"` AND `intervention ∈ control_markers`
2. For each cell type in training controls:
   - Compute per-gene mean μ and std σ
   - Apply shrinkage: σ_final = sqrt(σ² + ε), ε = 0.1
3. Apply to ALL cells (training + test, controls + interventions):
   - z = (x - μ_ct) / σ_ct
   - Clip to [-6, 6] to handle outliers
4. For unseen cell types: use global statistics from all training controls

**Result**: `adata.X` contains z-scored values; original in `adata.layers["X_log1p"]`

**Metadata**: All preprocessing details stored in `adata.uns["gcrl_preproc"]`

### 2. Intervention Parsing:

Extract TFs from `adata.obs['intervention']`:
- Parse strings: "TF1+TF2" → ["TF1", "TF2"]
- Validate: Each TF must exist in `var_names` with `kind == "TF"`
- Build `col_map`: {TF_name: index} for intervention vector construction

### 3. Community-Based Latent Structure:

```python
# From intervened TFs, identify unique communities
intervened_communities = unique(adata.var.loc[intervened_TFs, "community"])
p = len(intervened_communities)
z_dim = p + 1  # p communities + 1 global

# Hard mapping: TF → community → latent dimension
tf_to_latent[i] = community_to_latent[community_of_TF[i]]
```

### 4. Intervention Grouping:

Group intervened cells by intervention type for MMD loss:
```python
interv_groups = {
    "TF1": {"X": expression_matrix, "ct": cell_type_codes, "n_cells": count},
    "TF2": {"X": ..., "ct": ..., "n_cells": ...},
    "TF1+TF2": {"X": ..., "ct": ..., "n_cells": ...},
}
```

## Complete Loss Function (7 Components)

### Mathematical Formulation:

```
L_total = α_rec × L_reconstruction
        + β(t) × L_KL
        + λ_mcc × L_alignment
        + λ_sparse × L_sparsity
        + λ_dag × L_dag
        + α(t) × L_MMD
        + α(t) × λ_centroid × L_centroid  (optional, set to 0 to disable)
```

**Note**: The DAG acyclicity constraint (L_dag) is a new component that enforces the learned causal graph G is acyclic using the NOTEARS smooth constraint.

### 1. Reconstruction Loss (Controls Only):

**Formula**: `L_rec = MSE(x_rec, x_ctrl_all)`

**Purpose**: Learn to reconstruct control cell expression for ALL genes

**Data**: Only control cells from training set

**Weight**: α_rec = 1.0 (constant)

**Implementation**:
```python
# Input: TF expression only
mu, var = model.encode(x_ctrl_tf, ct=ct_vec)
z = model.reparameterize(mu, var)
# Output: All gene expression (TF + TG)
x_rec = model.decode(z, ct=ct_vec)
# Compare against all genes
loss_rec = F.mse_loss(x_rec, x_ctrl_all)
```

### 2. KL Divergence Loss (Scheduled):

**Formula**: `L_KL = -0.5 × mean(1 + log(σ²) - μ² - σ²)`

**Purpose**: Regularize latent space to N(0, I)

**Weight**: β(t) - Scheduled
- Epochs 1-10: β = 0
- Epochs 11+: Linear ramp from 0 to β_max = 0.01 (1e-2)

**Note**: Model returns variance σ² (NOT log-variance)
```python
mu, var = model.encode(x, ct=ct_vec)  # var = σ²
logvar = torch.log(var)               # Convert to log-variance
loss_kld = -0.5 * torch.mean(1 + logvar - mu.pow(2) - logvar.exp())
```

### 3. Eigengene Alignment Loss (Partial-MCC):

**Formula**: `L_mcc = partial_mcc_loss_torch(E, MU)`

**Purpose**: Align latent dimensions with TF community structure

**Data**: Control cells (same batch as reconstruction)

**Weight**: λ_mcc = 0.75 (constant, auto-configured based on number of communities)
**NOTE**: This value is CRITICAL for gCRL-VAE performance. Do not lower below 0.70.

**Eigengene Matrix E** (B × (p+1)):
- Columns 0..p-1: PC1 of TF expression within each community
- Column p: PC1 of all TFs (global eigengene)

**Latent Means MU**: First p+1 dimensions of encoder output μ

**Partial MCC**:
- Computes partial correlation between each (E_i, MU_i) pair
- Controls for correlations with other dimensions
- Returns negative mean partial correlation (minimize = maximize alignment)

**Implementation**:
```python
# Option 1: Use precomputed eigengenes
if "X_comm_eig" in adata_ctrl.obsm:
    E = adata_ctrl.obsm["X_comm_eig"][batch_idx, comm_columns]

# Option 2: Compute on-the-fly from TF expression
else:
    for community in comm_order:
        tf_in_comm = TFs with community == community
        E[:, i] = PC1(X_ctrl_batch[:, tf_in_comm])
    E[:, p] = PC1(X_ctrl_batch[:, all_TFs])

MU = mu[:, :(p+1)]
loss_mcc = partial_mcc_loss_torch(E, MU)
```

### 4. L1 Sparsity Loss (Causal Graph):

**Formula**: `L_sparse = ||G||_1` (full matrix, excluding diagonal)

**Purpose**: Encourage sparse causal graph structure

**Weight**: λ_sparse = 0.001 (constant, acts as tie-breaker)

**Schedule**: Active from epoch 10+ (sparse_start_epoch = 10)

**DAG Structure**:
- **UPDATED (2025-11-13)**: G is now a **full learnable matrix** (not constrained to upper-triangular)
- Acyclicity enforced via NOTEARS penalty (see Loss Component 5)
- L1 penalty reduced 10× (0.01 → 0.001) to act as gentle tie-breaker
- Late onset (epoch 10) allows early learning freedom

**Implementation**:
```python
loss_sparse = torch.tensor(0.0, device=device)
if cfg.lambda_sparse > 0 and epoch >= cfg.sparse_start_epoch:
    loss_sparse = torch.norm(model.G, p=1)  # L1 norm on full G (diagonal already zeroed)
    loss = loss + cfg.lambda_sparse * loss_sparse
```

### 5. NOTEARS DAG Acyclicity Constraint (Scheduled):

**Formula**: `L_dag = tr(exp(G ⊙ G)) - d`

**Purpose**: Enforce that the learned causal matrix G forms a directed acyclic graph (DAG)

**Weight**: λ_dag = 1.0 (strong penalty)

**Schedule**: Active from epoch 10+ (dag_start_epoch = 10)

**NOTEARS Method**:
- Smooth differentiable acyclicity constraint from Zheng et al. (2018)
- h(G) = tr(exp(G ⊙ G)) - d, where ⊙ is elementwise product, d is dimension
- h(G) = 0 if and only if G is acyclic
- Allows gradient-based optimization without combinatorial search
- Self-loops removed by zeroing diagonal before constraint

**Algorithm**:
```python
def notears_acyclicity(G: torch.Tensor) -> torch.Tensor:
    """
    NOTEARS-style smooth acyclicity constraint on the full adjacency matrix G.
    h(G) = tr(exp(G ⊙ G)) - d, where ⊙ is elementwise product.
    We remove self-loops from consideration by zeroing the diagonal.
    """
    # Remove self-loops
    G_offdiag = G - torch.diag_embed(torch.diag(G))
    # Elementwise square to ensure non-negativity as in original NOTEARS
    A = G_offdiag * G_offdiag
    exp_A = torch.matrix_exp(A)
    d = G.shape[0]
    h = torch.trace(exp_A) - d
    return h
```

**Implementation**:
```python
loss_dag = torch.tensor(0.0, device=device)
if cfg.lambda_dag > 0 and epoch >= cfg.dag_start_epoch:
    loss_dag = notears_acyclicity(model.G)
    loss = loss + cfg.lambda_dag * loss_dag
```

**Why NOTEARS?**
- Enables learning optimal DAG structure from data (no fixed ordering)
- More flexible than hard structural constraints (e.g., upper-triangular)
- Proven methodology from causal discovery literature
- Smooth and differentiable (compatible with gradient descent)

**Trade-offs**:
- More challenging optimization (larger search space)
- Matrix exponential is expensive for large latent dimensions
- Early epochs (1-9) have no DAG constraint for learning freedom
- May need more epochs to converge

### 6. MMD Loss (Intervention Validation, Scheduled):

**Formula**: `L_MMD = MMD(x_sim, x_real_rec)`

**Purpose**: Ensure simulated interventions match real perturbations

**Weight**: α(t) - Scheduled
- Epochs 1-5: α = 0
- Epochs 6+: Linear ramp from 0 to α_max = 1.0

**CRITICAL**: MMD computed on **ORIGINAL INTERVENED EXPRESSION**, NOT reconstructed

**Algorithm**:
```python
# 1. Sample random intervention type
interv_name = random_choice(["TF1", "TF2", ...])

# 2. Sample real intervened cells (all genes) - ORIGINAL expression
X_real_all = sample(intervened_cells[interv_name], n=batch_size//2)

# 3. Sample control cells (TF-only)
X_ctrl_sample_tf = sample(control_cells[:, tf_idx], n=batch_size//2)

# 4. Build intervention vector
c = one_hot_encode(interv_name, n_TFs)  # Binary vector indicating which TFs

# 5. Simulate intervention on controls
mu_ctrl, var_ctrl = model.encode(X_ctrl_sample_tf, ct=ct_vec)
z_ctrl = model.reparameterize(mu_ctrl, var_ctrl)
u_sim = model._dag_transform(z_ctrl, bc, csz, None, None)  # Apply intervention
x_sim = model.decode(u_sim, ct=ct_vec)  # Decode to all genes (SIMULATED)

# 6. Real intervened cells: use ORIGINAL expression directly
# No need to encode-decode; we already have the ground truth!

# 7. Compute MMD: simulated vs. original real expression
loss_mmd = MMD_loss(x_sim, X_real_all)
```

**MMD Implementation** (Multi-scale Gaussian Kernel):
- Adaptive bandwidth: σ² = sum(distances) / (n² - n)
- Multi-scale: 5 kernels with bandwidth scaling by factor 2.0
- Kernel: k(x, y) = exp(-||x - y||² / σ²)
- MMD²: E[k(x, x')] + E[k(y, y')] - 2E[k(x, y)]

**Why Original Expression (not reconstructed)?**
- Real intervened cells are the ground truth
- No need to add reconstruction noise
- Direct comparison: simulated prediction vs. actual outcome
- Both in expression space (same units, same scale)

### 7. Centroid Loss (Optional):

**Formula**: `L_centroid = MSE(mean(x_sim), mean(X_real_all))`

**Purpose**: Enforce mean shift matching between simulated and real interventions

**Weight**: λ_centroid = 0.1 (default, set to 0 to disable)
**Schedule**: Uses **same schedule as MMD** (α(t) from epochs 5+)

**Implementation**:
```python
# Computed during MMD loss calculation
centroid_sim = x_sim.mean(dim=0)          # Mean across batch (gene-wise means)
centroid_real = X_real_all.mean(dim=0)    # Mean of ORIGINAL real expression
loss_centroid = F.mse_loss(centroid_sim, centroid_real)

# Apply same schedule as MMD for consistency
loss = loss + alpha_mmd * lambda_centroid * loss_centroid
```

**Why Add Centroid Loss?**
- Directly targets mean expression shifts (key biological effect)
- Lower variance than full distribution matching
- Complements MMD which captures full distribution
- Optional - can be disabled if redundant

**Why Same Schedule as MMD?**
- Both components validate intervention effects
- Consistent ramping prevents one from dominating early
- Simpler hyperparameter tuning

**When to Use:**
- Enable (λ > 0) for datasets with strong mean shift effects
- Disable (λ = 0) if MMD alone is sufficient
- Recommended starting value: 0.1

## Training Loop

### Initialization:

```python
# Loss schedules
beta_schedule[epoch]:
    Epochs 1-10: 0
    Epochs 11+: Linear ramp 0 → β_max

alpha_schedule[epoch]:
    Epochs 1-5: 0
    Epochs 6+: Linear ramp 0 → α_max

temp_schedule[epoch]:
    Linear increase 1.0 → 10.0 (for soft routing, if used)

# Optimizer
Adam(lr=2e-3)

# Gradient clipping
clip_norm = 0.5
```

### Per-Batch Training:

```python
for epoch in 1..n_epochs:
    beta_kl = beta_schedule[epoch]
    alpha_mmd = alpha_schedule[epoch]
    temp = temp_schedule[epoch]

    for batch in control_batches:  # Controls only, single cell-type batches
        # === Forward pass ===
        x = batch["x"]  # Control expression
        ct_vec = one_hot(batch["ct"])  # Cell-type if ct_dim > 0

        mu, var = model.encode(x, ct=ct_vec)
        z = model.reparameterize(mu, var)
        x_rec = model.decode(z, ct=ct_vec)

        # === Loss 1: Reconstruction ===
        loss_rec = MSE(x_rec, x)

        # === Loss 2: KL Divergence (scheduled) ===
        logvar = log(var)
        loss_kld = -0.5 × mean(1 + logvar - mu² - exp(logvar))

        # === Loss 3: Eigengene Alignment ===
        E = compute_eigengenes(batch)  # (B, p+1)
        MU = mu[:, :(p+1)]
        loss_mcc = partial_mcc_loss(E, MU)

        # === Loss 4: L1 Sparsity (epoch 10+) ===
        if epoch >= 10:
            loss_sparse = ||model.G||_1

        # === Loss 5: DAG Acyclicity (NOTEARS, epoch 10+) ===
        if epoch >= 10:
            loss_dag = notears_acyclicity(model.G)

        # === Loss 6: MMD (scheduled, epoch 5+) ===
        if alpha_mmd > 0:
            # Sample intervention
            interv = random_intervention()
            X_real = sample_intervened_cells(interv)
            X_ctrl_sample = sample_controls()
            c = build_intervention_vector(interv)

            # Simulate vs. Real
            u_sim = model(X_ctrl_sample, c=c, ...)
            u_real = model(X_real, c=c, ...)
            loss_mmd = MMD(u_sim, u_real)

        # === Total Loss ===
        loss = α_rec × loss_rec +
               beta_kl × loss_kld +
               λ_mcc × loss_mcc +
               λ_sparse × loss_sparse +
               λ_dag × loss_dag +
               alpha_mmd × loss_mmd +
               alpha_mmd × λ_centroid × loss_centroid

        # === Optimization ===
        loss.backward()
        clip_grad_norm_(model.parameters(), 0.5)
        optimizer.step()
```

## Key Implementation Details

### 1. Cell-Type One-Hot Encoding:
```python
if ct_dim > 0:
    ct_vec = F.one_hot(ct_batch.long(), num_classes=ct_dim).float()
    # Shape: (batch_size, ct_dim)
else:
    ct_vec = None
```

### 2. Variance vs. Log-Variance:
**IMPORTANT**: Model returns variance σ², NOT log-variance
```python
mu, var = model.encode(x, ct)  # var = σ²
logvar = torch.log(var)         # Convert for KL formula
```

### 3. DAG Transform:
```python
# In model.forward():
z = reparameterize(mu, var)

# Apply intervention shift
bc = hard_routing(c, tf_to_latent)  # Route to specific latent dim
csz = c @ c_shift                    # Intervention magnitude

# Propagate through DAG (UPDATED 2025-11-13: Full G matrix)
G_offdiag = G - diag_embed(diag(G))  # Remove diagonal (no self-loops)
A = I - G_offdiag                     # Full matrix (not just upper triangle)
Ainv = inv(A)                         # Acyclicity enforced via NOTEARS penalty
u = (z + bc * csz) @ Ainv.T
```

**Key Change**: G is now a **full learnable matrix** with acyclicity enforced via NOTEARS penalty, not structural constraint.

### 4. Intervention Vector Construction:
```python
# For intervention "TF1+TF2"
c = torch.zeros(batch_size, n_intervened_TFs)
c[:, tf_index["TF1"]] = 1.0
c[:, tf_index["TF2"]] = 1.0
```

### 5. Eigengene Computation (On-the-Fly):
```python
def compute_pc1(X):
    """Power iteration for first principal component"""
    Xc = X - X.mean(dim=0)
    v = random_vector()
    for _ in range(12):  # Power iterations
        v = Xc @ (Xc.T @ v)
        v = v / (v.norm() + 1e-8)
    scores = Xc @ v
    return z_score(scores)
```

## GPU Memory Management and Batch Size Selection

**UPDATED (2025-11-23)**: Intelligent GPU memory management with empirically calibrated estimation.

### Overview

The training schema includes automatic batch size selection based on available GPU memory to prevent out-of-memory (OOM) errors. This is critical for datasets with large gene counts (>2500 genes) where the polynomial decoder can consume significant GPU memory.

### Memory Estimation

The system estimates GPU memory requirements accounting for all major components:

1. **Model Parameters**: Encoder + polynomial decoder (linear + quadratic terms)
2. **Optimizer States**: Adam maintains 2 states (momentum + variance) per parameter
3. **Forward Pass Activations**: All intermediate tensors stored for backpropagation
4. **Backward Pass Gradients**: ~3x forward pass memory (polynomial decoder is gradient-intensive)
5. **MMD Loss Kernel Matrices**: Quadratic in batch size: (2×batch_size)² for multi-scale kernels
6. **Eigengene Computation**: Covariance matrices for partial-MCC alignment
7. **DAG Transform**: Matrix inversions and latent propagations
8. **PyTorch Memory Overhead**: Allocator fragmentation and autograd graph

### Empirical Calibration

**Critical Finding**: The polynomial decoder + PyTorch autograd consumes **50-60x more memory** than naive calculations suggest for large gene counts!

**Example (Norman2019: 5000 genes, 21.2 GB GPU)**:
- Naive calculation: 0.20 GB
- Real usage: ~10-12 GB
- Required overhead factor: **50x**

This massive overhead comes from:
- PyTorch's autograd graph storing all intermediate tensors
- Polynomial decoder quadratic terms creating huge intermediate matrices
- Memory fragmentation (PyTorch allocator can't reuse fragmented blocks)
- No gradient checkpointing (would reduce memory but slow training)

### Overhead Scaling

The system uses gene-count-aware overhead factors:

```python
base_overhead = 10.0  # 10x minimum (empirically validated)

# Additional scaling for large gene counts
if n_genes > 4000:
    gene_factor = 5.0   # Total 50x overhead
elif n_genes > 2500:
    gene_factor = 3.5   # Total 35x overhead
elif n_genes > 1500:
    gene_factor = 2.0   # Total 20x overhead
else:
    gene_factor = 1.0   # Total 10x overhead
```

### Automatic Batch Size Selection

The `auto_select_batch_size()` function uses binary search to find the optimal batch size:

1. **Query GPU memory**: Get total VRAM from `torch.cuda.get_device_properties()`
2. **Set target utilization**: Default 35% of GPU memory (conservative but not wasteful)
3. **Binary search**: Find largest batch size (power of 2) that fits within target
4. **Minimum batch size**: 16 (ensures reasonable training efficiency)

**Example**:
```
GPU: 21.2 GB available
Target: 35% = 7.4 GB

Binary search:
  batch_size=512 → 10.0 GB (too high)
  batch_size=384 → 7.51 GB (too high)
  batch_size=256 → 5.06 GB (✓ selected)
  batch_size=128 → 2.63 GB (safe but slower)
```

### Dataset-Aware Selection

The system combines two strategies:

1. **Ideal batch size** (based on dataset characteristics):
   - <100 controls/cell-type: 64
   - <500 controls/cell-type: 128
   - <2000 controls/cell-type: 256
   - ≥2000 controls/cell-type: 512

2. **Memory-aware batch size** (based on GPU constraints)

**Final batch size** = min(ideal_batch_size, memory_aware_batch_size)

### Runtime OOM Protection

Even with careful estimation, OOM errors can occur due to:
- Memory fragmentation varying between runs
- Other processes using GPU memory
- PyTorch's internal memory management

The training loop includes exception-based OOM handling:

```python
try:
    # Normal training step
    loss = compute_loss(model, batch)
    loss.backward()
    opt.step()

except RuntimeError as e:
    if "out of memory" in str(e).lower():
        # Clear cache and skip batch
        torch.cuda.empty_cache()
        opt.zero_grad(set_to_none=True)

        # Log detailed diagnostics
        # Continue training (up to 5 OOM events)
```

**OOM Handling Strategy**:
- **Zero overhead** during normal operation (try/except is free when not triggered)
- **Graceful degradation**: Skip problematic batches, continue training
- **Detailed diagnostics**: Reports batch size, gene count, GPU info
- **Automatic termination**: Stop after 5 OOM events with actionable recommendations

### Configuration Output

When running `analyze_dataset_and_suggest_config()`, the system reports:

```
⚙️  Suggested Configuration:
  - GPU Memory: 21.2 GB available
    → Target usage: 35% (7.4 GB) for safe training
  - Batch size: 256 (GPU memory limited)
    → Ideal from dataset: 512
    → GPU memory allows: 256
    → Selected: 256 (conservative)
    → Estimated memory: 5.1 GB
```

If memory usage is high (>8 GB estimated), the system provides recommendations:
1. Reduce genes: Keep top 2000-3000 highly variable genes
2. Further reduce batch size
3. Focus gene selection on GRN (TFs + TF targets + HVGs)

### Manual Override

Users can override the automatic selection:

```python
cfg = analyze_dataset_and_suggest_config(adata, outdir="...", verbose=True)
cfg.batch_size = 384  # Manual override
model, history = train_gcrl_vae(adata=adata, cfg=cfg)
```

The runtime OOM protection will catch any issues.

### Recommendations

**For datasets with >4000 genes**:
- Expect batch_size reduction from ideal (512 → 128-256)
- Consider reducing to 2500-3000 HVGs for better memory efficiency
- Monitor first few epochs for OOM events
- If OOM persists, reduce gene count rather than batch size

**For datasets with <2000 genes**:
- Memory constraints rarely binding
- Batch size determined primarily by dataset size
- Can often use larger batch sizes (512-1024)

## Differences from Discrepancy-VAE

| Feature | Discrepancy-VAE | gCRL-VAE |
|---------|-----------------|----------|
| **Reconstruction target** | All cells | Controls only |
| **Batching** | By intervention | By cell type (controls) |
| **Additional losses** | None | Eigengene alignment (partial-MCC) |
| **Latent structure** | General | Community-based (p+1) |
| **Cell-type conditioning** | Not used | Required for multi-CT |
| **Preprocessing** | Standard | Per-CT controls-only z-scoring |
| **MMD computation** | On decoded expression | On DAG-transformed latents |

## Common Issues & Solutions

### Issue 1: "Intervention TF not found"
**Cause**: TF in `intervention` column not in `var_names` or not marked `kind == "TF"`
**Solution**: Ensure all intervened TFs are in gene set with correct annotation

### Issue 2: "MU/E shape mismatch"
**Cause**: Eigengene matrix doesn't match latent dimensions
**Solution**: Check that p communities are correctly identified and eigengenes computed

### Issue 3: MMD loss is NaN
**Cause**: No intervened cells available for MMD computation
**Solution**: Ensure training set contains both controls and interventions

### Issue 4: KL loss explodes or dominates
**Cause**: Beta schedule might be incorrect
**Solution**: Use proven value `beta_kld_max=0.01` (1e-2), NOT 1e-3 or 1e-4
**Note**: The proven optimal value is 0.01, which provides gentle regularization

## Hyperparameter Recommendations

### Default Configuration (Proven Optimal - v1):
```python
VAEConfig(
    batch_size=512,       # For Norman2019 (8907 cells); use 256 for smaller datasets
    epochs=50,
    lr=2e-3,

    # Loss weights (constant) - PROVEN OPTIMAL VALUES
    alpha_rec=1.0,        # ✅ Balanced reconstruction weight
    lambda_mcc=0.75,      # ✅ CRITICAL: Auto-configured for community structure
                          #    (Norman2019 has 6 communities → 0.75)
                          #    DO NOT LOWER BELOW 0.70!
    lambda_sparse=0.001,  # Sparsity on causal graph G (reduced from 0.02, acts as tie-breaker)
    lambda_dag=1.0,       # DAG acyclicity penalty (NOTEARS)
    dag_start_epoch=10,   # When to start DAG penalty
    sparse_start_epoch=10,# When to start L1 sparsity
    lambda_centroid=0.05, # Centroid loss (set to 0 to disable)
                          # NOTE: Uses same schedule as MMD (alpha_mmd)

    # Loss weights (scheduled) - PROVEN OPTIMAL VALUES
    beta_kld_max=0.01,    # ✅ Gentle KL regularization (1e-2)
                          #    This is 10× the old default of 1e-3
    alpha_mmd_max=1.0,    # Also applies to centroid loss

    # MMD kernel
    mmd_kernel_mul=2.0,
    mmd_kernel_num=5,

    # Preprocessing
    preproc_epsilon=0.1,
    preproc_clip_value=6.0,
)

GCRLVAEConfig(
    input_dim=n_TFs,              # TF-only input
    output_dim=n_genes,           # All genes output
    hidden_enc=(128,),            # Simplified encoder
    use_polynomial_decoder=True,  # Polynomial decoder
)
```

**IMPORTANT**: These values (alpha_rec=1.0, lambda_mcc=0.75, beta_kld_max=0.01) are the
**proven optimal configuration (v1)** that dominated alternative configurations across
multiple metrics. DO NOT change these without extensive validation.

### Tuning Guidelines:

**For small datasets (<1000 cells)**:
- Reduce batch_size to 128 or 256
- Keep beta_kld_max at 0.01 (proven optimal)
- Can increase lambda_sparse to 0.002-0.005 (gentle sparsity encouragement)
- Keep lambda_dag at 1.0 (strong DAG enforcement)
- Keep simplified encoder (128 hidden units)
- ⚠️ DO NOT change lambda_mcc without validation

**For large datasets (>10000 cells)**:
- Increase batch_size to 512 (as in Norman2019)
- Train for more epochs (100+) if convergence is slow
- ⚠️ DO NOT reduce lambda_mcc below 0.70 - this was proven to harm performance
- Can increase encoder size to (256,) if needed

**For many cell types (>5)**:
- Ensure ct_dim is set correctly
- May need larger encoder: hidden_enc=(256,) or (128, 128)

**For weak intervention effects**:
- Increase alpha_mmd_max to 2.0
- Start MMD earlier (epoch 3 instead of 5)
- Increase lambda_centroid to 0.2 or 0.5

**For strong intervention effects**:
- Can disable centroid loss (lambda_centroid=0.0)
- MMD alone may be sufficient
