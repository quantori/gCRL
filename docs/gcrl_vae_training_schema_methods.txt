### gCRL-VAE: Causal Variational Autoencoder with Gene Regulatory Network Priors

Updated 2026-05-01. Previous version saved as
gcrl_vae_training_schema_methods_backup_2026_04_30.txt.

#### Architecture and Causal Structure

The gCRL-VAE architecture extends the discrepancy-VAE (CMVAE, Shen et al. 2021) with two
optional GRN-informed components, controlled by boolean flags use_tf_only and
use_GRN_priors in both VAEConfig and GCRLVAEConfig.

The encoder maps input expression x_enc ∈ ℝ^input_dim to distributional parameters via a
single hidden layer: fc1(input_dim → hidden_enc) → LeakyReLU(0.2) → (fc_mean, fc_var),
producing (μ, σ²). When use_tf_only=True (default), input_dim = n_TF and only TF
expression is provided to the encoder; when False, input_dim = n_genes and the full gene
expression is used. The decoder is identical to CMVAE: d1(z_dim → 128) → LeakyReLU(0.2)
→ d2(128 → n_genes). Output always covers all genes (TF + TG).

From the encoder output, an observational latent z ~ N(μ, σ²) is sampled via the
reparameterization trick. The intervention shift is computed as δ = bc ⊙ (c @ c_shift),
where bc ∈ ℝ^z_dim is the routing vector and c_shift ∈ ℝ^c_dim is a learned per-TF
magnitude parameter. When use_GRN_priors=True, bc is computed via hard routing driven by
a learned permutation π (described below). When use_GRN_priors=False, soft routing is
used: a small MLP (c_head: c_dim → hidden_enc → z_dim) followed by softmax produces bc,
reproducing CMVAE's c_encode. The DAG transform is u = (z + bc * csz) @ inv(I − G_triu),
where G_triu = triu(G, diagonal=1) and G ∈ ℝ^(z_dim × z_dim) is a learnable full matrix
with only its strictly upper-triangular part used, enforcing a fixed causal ordering over
latent dimensions. The intervened latent u is decoded to predicted perturbed expression
y_hat.

The latent space has dimension z_dim = p + 1, where p is the number of unique TF
communities among intervened TFs, plus one global dimension. This structure is derived
automatically from adata.var["community"] at training time.

#### Permutation Learning (use_GRN_priors=True)

A square score matrix S ∈ ℝ^(z_dim × z_dim) is learned as an nn.Parameter, initialized
to the identity matrix. At each forward pass the hard permutation is derived as:

    π[k] = argmax(S[k, :])     for k = 0, …, z_dim-1

π simultaneously serves three roles:

1. Routing. TF with community index k is routed to latent dimension π[k], making the
   routing assignment data-driven rather than fixed at initialization. This replaces the
   previous fixed tf_to_latent lookup array.

2. Causal ordering. The DAG upper triangle triu(G, diagonal=1) enforces acyclicity by
   imposing a causal order over latent dimensions. With the learned π, this ordering is
   also learned: latent dimension π[0] is the most upstream cause, π[1] next, and so on.
   The model therefore learns which community is causally upstream of which, rather than
   inheriting an arbitrary fixed order.

3. MCC alignment. The eigengene matrix E is reordered by π at loss time so that eigengene
   column π[k] is compared against latent dimension k:
       partial_mcc_loss_torch(E[:, π], μ)
   No additional linear alignment matrix is required.

π is non-differentiable (argmax), but S is updated by gradient descent because the MCC
loss value changes as the implied column assignments change. Gradients flow through the
MCC values into μ and from there into S. S is initialized to the identity so that
training starts from the trivial permutation (community k → latent k), avoiding an extra
source of randomness at the start of training.

#### Data Preparation

Input data is stored as AnnData objects with normalized and log₁₊ₓ-transformed expression
values in adata.X (float32, possibly sparse). No further preprocessing is applied inside
the training function — adata.X is used as-is, matching discrepancy-VAE behaviour. Gene
metadata includes kind ("TF" or "TG") and community (TF community assignment); cell
metadata specifies cell_type, intervention, and set (training/test). The eigengene matrix
adata.obsm["X_comm_eig"] of shape (n_cells, ≥ p+1) must be precomputed and stored before
calling train_gcrl_vae() when use_GRN_priors=True; the training function raises a KeyError
at startup if the key is absent. The first p+1 columns of the eigengene matrix are used;
their order relative to latent dimensions is handled by π at loss time.

Intervened TFs are parsed from the intervention column, validated against gene annotations,
and mapped to a binary intervention vector c ∈ {0,1}^c_dim and to community indices via
adata.var["community"]. The community indices (tf_comm_ids) and the final learned
permutation (perm) are saved to eval_wiring.json.

#### Training Strategy

Training follows discrepancy-VAE exactly. A _PairedDataset pairs each intervened cell with
a randomly pre-sampled control cell, storing tuples (x_ctrl_enc, y_interv, c_binary,
x_ctrl_all, eig) per item — the same structure as SCDataset in the original
implementation. When use_GRN_priors=True, the eigengene row for the same sampled control
cell is also stored (self.E), ensuring that E[i] and mu[i] in the loss always correspond
to the same cell. An InterventionBatchSampler (analogous to SCDATA_sampler) groups cells
from the same intervention condition into each batch. Incomplete batches (size < batch_size)
are dropped.

The model is trained on the training split only. No validation set is used and no early
stopping is applied. The best model is saved by training loss (MMD + MSE + KLD + L1),
matching discrepancy-VAE's checkpointing strategy. The last-epoch model is also saved.

#### Loss Function

The training objective is:

    L = α(t) · MMD(y_hat, y_interv)
      + MSE(x_recon, x_ctrl_all)
      + β(t) · KLD(μ, σ²)
      + λ_sparse · ||triu(G, diagonal=1)||₁
      [+ λ_mcc · partial_mcc_loss(E[:, π], μ)]   when use_GRN_priors=True

The MMD loss, reconstruction MSE, KLD, and L1 sparsity terms are taken verbatim from
discrepancy-VAE's loss_function(). The MMD kernel employs multi-scale Gaussian kernels
with adaptive bandwidth: given the combined (2n × d) sample matrix, pairwise squared
Euclidean distances are computed and bandwidth set as σ² = Σ D_ij / (n² - n). Five kernel
scales with bandwidth multiplier 2.0 are summed. The encoder returns variance σ² (not
log-variance); KLD is computed as −0.5 · mean(1 + log σ² − μ² − σ²).

Perturbed cells contribute only to the MMD term. The reconstruction MSE is computed
between x_recon (the zero-intervention decoded output, i.e. num_interv=0 path) and
x_ctrl_all (the original full-gene expression of the sampled control cell). Perturbed
cells are not passed through the encoder-decoder and have no per-cell reconstruction loss.

The MCC alignment loss (use_GRN_priors=True only) is partial_mcc_loss_torch(E[:, π], μ),
where E ∈ ℝ^(B×(p+1)) contains the precomputed eigengene rows for the control cells in
the current batch, and π is the hard permutation derived from S. The reordering E[:, π]
places eigengene column π[k] alongside latent dimension k before computing partial
correlations. The partial-MCC loss computes, for each column k, the partial correlation
between E[:, π[k]] and μ[:, k] conditioned on all other columns, then returns the negative
mean. Minimizing this loss aligns each latent dimension one-to-one with the eigengene of
its assigned community, without assuming any a priori column correspondence.

Loss schedules follow discrepancy-VAE: β(t) = 0 for epochs 0–9, then linear ramp to
β_max; α(t) = 0 for epochs 0–4, linear ramp to α_max for epochs 5 to epochs//2, plateau
at α_max thereafter. The softmax temperature τ(t) = 1.0 for epochs 0–4, then linear ramp
to temp_max (used only when use_GRN_priors=False).

#### Optimization

All parameters — encoder, decoder, G, c_shift, S (if GRN priors), and c_head (if soft
routing) — are optimized jointly by a single Adam optimizer (lr=1e-3, default). S is
included in model.parameters() automatically because it is an nn.Parameter registered in
__init__. Gradient norm clipping (max norm 0.5) is applied at each step.

#### Prediction

Post-training prediction is performed by GCRLVAE.predict(adata), which generates an
AnnData object with predicted expression for all (cell_type, intervention) pairs in the
test set. For each group, n_pred control cells are sampled from training controls (with
replacement), encoded via the TF-only (or all-gene) encoder, and the intervention is
applied via the DAG transform followed by decoding. Routing uses the final learned π from
S. The returned AnnData has obs columns: set ("prediction"), cell_type, intervention,
test_cell_idx (obs index of the corresponding real test cell), and control_cell_idx (obs
index of the control cell used as seed). Double perturbations are handled by applying two
additive DAG shifts (bc1*csz1 + bc2*csz2), consistent with discrepancy-VAE's
double-intervention logic.

#### Comparison to Discrepancy-VAE and Previous gCRL-VAE Versions

Relative to discrepancy-VAE (CMVAE), gCRL-VAE adds when use_GRN_priors=True: (1) hard
community-based routing via learned permutation π replacing the soft MLP routing, and
(2) the MCC alignment loss term partial_mcc_loss(E[:, π], μ) with learnable score matrix
S. When use_tf_only=True, the encoder receives only TF expression rather than all-gene
expression. All other components (decoder, DAG transform, MMD loss, schedules,
checkpointing) are identical to CMVAE.

Relative to the 2026-04-30 gCRL-VAE version, the current implementation replaces the
learnable linear alignment map X_align (z_dim × p+1) with the score matrix S (z_dim ×
z_dim) and its derived hard permutation π. The key difference is that X_align could learn
arbitrary linear rotations and scalings — it aligned projections of μ onto eigengene
space, but with no guarantee of one-to-one correspondence between latent dimensions and
eigengenes. Under the new formulation, π enforces a strict one-to-one assignment: each
latent dimension is compared directly to exactly one eigengene column, with no free
linear mixing. Additionally, π now determines the causal ordering of the DAG, making
the routing assignment and causal structure jointly learned rather than fixed arbitrarily
at initialization.
